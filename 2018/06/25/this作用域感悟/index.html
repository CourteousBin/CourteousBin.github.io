javascript 中 this 感悟

Javascript 中 this 的指向一直是困扰我很久的问题，在使用中出错的机率也非常大。

在面向对象语言中，它代表了当前对象的一个引用，在 javascript 它不是固定不变的，而是随着它的执行环境的改变而改变。

在 Javascript 中 this 总是指向 调用它所在方法的对象。因为this是在函数运行时，自动生成的一个内部对象，只能在函数内部使用。

默认 this 指向全局(window)

默认环境下 , this 指向全局

因为函数里面的 this 指向全局，如果在函数内部修改 name , 外部的 name 会改变吗？

正因为是指向全局，所以，外部的全局变量发生了改变。

在对象中 this 指向该调用对象

如果函数作为对象的方法调用，this 指向的是这个上级对象，即调用方法的对象。

构造函数中的this指向新创建的对象本身

构造函数的 this 与 对象的 this 基本一致，谁调用构造函数，this 就 指向它本身。

上述代码中，我们通过 new 关键字创建一个对象的实例，new 关键字可以改变 this 的指向，将这个 this 指向对象 obj 。

在我们新实例化一个对象时，javascript帮我们做了几件事？

  • 创建空对象

  • 调用构造函数
  • 返回新对象的地址
