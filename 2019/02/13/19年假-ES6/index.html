深度拷贝

理解:

  • 基本类型
  • 引用类型
  • 基本类型的数据是存放在栈内存中的，而引用类型的数据是存放在堆内存中的

基本类型的复制就是在栈内存中开辟出了一个新的存储区域用来存储新的变量，这个变量有它自己的值，只不过和前面的值一样，所以如果其中一个的值改变，则不会影响到另一个。

复制一个对象，改变值，为什么会同时发生变化？

定义了一个对象其实是在栈内存中存储了一个指针，这个指针指向堆内存中该对象的存储地址。复制给另一个对象的过程其实是把该对象的地址复制给了另一个对象变量，两个指针都指向同一个对象，所以若其中一个修改了，则另一个也会改变。

箭头函数

最简单的箭头函数

如果只有一个参数,可以省略小括号 ()
如果只有返回值,没有函数体,则可以胜率 {}

this 指向

解决:

箭头函数没有自己的this , 他会用上层的this

测试题:

  • 有何不同?

答: obj1 getNamge(){} 这种写法只是对象里面方法的简写,与 getName:function(){} 没有任何区别,输出 obj ;
obj2 getName:()=>{} 是用了箭头函数方式,用这种方式会改变 this 指向,也就是说, Obj2 getName 此时的 this 指向外层 (window) , 调用 obj2.getName() 输出 undefined;

  • 输出什么?

答: undefined,obj因为用箭头函数指向外层 window 所以 undefined，无论谁调用都是一个结果。

  • 输出什么?

答: 输出 obj2 。在 obj getName(){} 定义方法时 并没有改变 this 指向 , this 指向调用它的本身。 obj2 调用 obj.getName 时 , 其实等于偷懒的简写 , this 也是指向调用它的本身 , 就是 obj2 ,所以会输出 obj2

总结:

  • 箭头函数虽然好,但是不能滥用,不能应用于所有场景
  • 如果需要 this 随着调用者变化而发生变化的话 , 就不适合使用箭头函数
  • 如果需要引用外层 this 的话就可以使用

Promise

基础

基础知识:

  • Promise 是一个类/构造函数
  • 可以创建实例
  • Promise 承诺 用于异步任何
  • promise的三种状态
    • pending(初始状态)
    • fulfilled(成功状态)
    • rejected(失败状态)
    • 注意promsie状态 只能由 pending => fulfilled/rejected, 一旦修改就不能再变
  • promise对象方法
    • then方法注册 当resolve(成功)/reject(失败)的回调函数
    • then方法是异步执行的
    • promise.then(onFulfilled, onRejected);
